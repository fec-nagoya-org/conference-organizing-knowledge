# カンファレンス開催ノウハウ

このスキルは、日本のテックカンファレンス主催者コミュニティで蓄積された開催ノウハウを提供する。
コロナ禍でリセットされたノウハウの継承を目的として、有志がまとめた実践的な知見集である。

出典: https://docs.google.com/document/d/1b0GAEADYCefXw4GZNTO6mktZDhZ-hno7QIziqh54W4I

## ナレッジの構成

詳細なナレッジは `references/` ディレクトリにトピックごとに分割して格納されている。
ユーザーの質問に応じて、該当するリファレンスを参照して回答すること。

リファレンス一覧:
- カンファレンス開催ノウハウ: ドキュメントの趣旨、編集方針
- スケジュール感: 開催日決定、チケット販売、CFP募集、スポンサー募集のタイミング
- 会場: 入館管理、設備確認、申請ルール、レイアウト
- スポンサー: パッケージ設計、対応フロー、トラブル事例
- 法人格と口座: 法人化、任意団体運営
- イベント中止と保険: 中止ポリシー、興行中止保険
- チケット: プラットフォーム選定、価格設定、抽選
- セッションの構成: セッション形式、パネルディスカッション、LT
- タイムテーブル: 時間配分、休憩設計、部屋割り
- CFP: Call for Paper運営、選考基準
- 登壇者の招致: 招待講演、海外登壇者
- グッズ作成: Tシャツ、ノベルティ
- 配信: ライブ配信、アーカイブ
- 同時通訳: 通訳手配、機材
- スタッフ募集: 募集、マネジメント、当日配置
- サイト作成: ドメイン、イベントサイト
- ケータリング: 発注、廃棄、アレルギー対応
- Wi-Fi: NOC構築、機材調達、敷設
- デザイン: ロゴ、LP、著作権
- 予算感: 実際のカンファレンスの予算事例
- トラブル事例集: 過去の失敗と教訓
- Code of Conduct: 策定背景、掲載事項、運用
- ブース: ブース設計、運営

## 回答の方針

- ユーザーの質問に対して、該当するセクションの内容を要約して回答する
- 具体的な事例（Vue Fes Japan、RubyKaigi、TSKaigi等）がある場合は参考として提示する
- スケジュールに関する質問には、逆算の考え方を含めて回答する
- トラブル防止の観点から、見落としがちな注意点も併せて伝える
